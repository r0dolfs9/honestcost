Integration snippets.

Four code blocks, copy-paste-ready, each with the exact insertion point in your current index.html and a tiny preview of the rendered UI. Variable names match the live file (renderResults, lastA, lastG).

One non-negotiable rule

Every snippet here belongs inside or below the result screen — never on the input form, never above the fold. The moment the tool nudges a user before showing them numbers, trust is gone and so is the dealer pitch.

02 / list-building · live week 1Email capture (EmailJS)

One unobtrusive block at the bottom of the result screen. Sends the user their comparison URL + a text summary. You receive an email yourself with each capture — that's your CRM until volume justifies a real one.

Setup, once

  1. Create an EmailJS account (free tier = 200 emails/month).
  2. Connect a Gmail / outlook service. Note the service_id.
  3. Create a template with variables {{to_email}}, {{summary}}, {{share_url}}. Note the template_id.
  4. Copy your public_key from the account settings.
  5. Paste below — replace the three placeholders.

function sendCap(e){
  e.preventDefault();
  const status = document.getElementById('emailCapStatus');
  const email  = document.getElementById('capEmail').value;
  const share  = location.origin + location.pathname + location.hash;
  const summary = `${lastA.name} pret ${lastB.name}, ${lastG.yrs} g., ${lastG.km} km/g.`;
  status.textContent = 'Sūta…';
  emailjs.send('YOUR_SERVICE_ID','YOUR_TEMPLATE_ID',{
    to_email: email, share_url: share, summary: summary,
  }).then(()=>{
    status.textContent = '✓ Saite ir tavā e-pastā.';
    plausible('email_captured');
    document.getElementById('emailCapForm').reset();
  }).catch(()=>{ status.textContent = 'Kaut kas nogāja greizi. Pamēģini vēlreiz.'; });
}

04 / instrumentation · live day 1Plausible custom events

Plausible's free tier supports custom events without extra cost. Drop these into the right handlers, then create matching "Goals" in your Plausible dashboard to turn them into conversion funnels.

snippet/events.js
scatter through your existing handlers 
// Add this once near the top of your script — fallback if Plausible is blocked
window.plausible = window.plausible || function(){ (window.plausible.q = window.plausible.q || []).push(arguments) };

// ─── 1 · inside doAnalyse(), after results render ───
plausible('analysis_run', { props: {
  car_a:   carA.id || carA.name,
  car_b:   carB.id || carB.name,
  horizon: g.yrs,
  km:      g.km,
  winner:  rA.totalMo < rB.totalMo ? 'A' : 'B',
  savings: Math.round(Math.abs(rA.total - rB.total))
}});

// ─── 2 · inside example button click handlers ───
plausible('example_loaded', { props: { preset: 'bmw_118i_x1' } });

// ─── 3 · inside the "Kopēt saiti" button handler ───
plausible('share_link_copied');

// ─── 4 · inside CAR_DB search input ───
let _searchTimer;
document.getElementById('carSearchA').addEventListener('input', (e) => {
  clearTimeout(_searchTimer);
  _searchTimer = setTimeout(() => {
    if (e.target.value.length >= 3)
      plausible('car_db_search', { props: { query: e.target.value.toLowerCase() } });
  }, 800);
});

// ─── 5 · inside ownership horizon buttons (3g / 5g / 7g / 10g) ───
plausible('horizon_changed', { props: { years: y } });

// ─── 6 · scroll-depth on results (one-time per session) ───
let _depthFired = false;
window.addEventListener('scroll', () => {
  if (_depthFired) return;
  const doc = document.documentElement;
  if ((doc.scrollTop + window.innerHeight) / doc.scrollHeight > 0.85) {
    plausible('scrolled_full_result');
    _depthFired = true;
  }
});

Goals to configure in Plausible dashboard

For each custom event below, create a matching Goal in Plausible → Site Settings → Goals → Add Goal → Custom Event:

  • analysis_run — the core funnel-conversion event
  • share_link_copied — virality proxy
  • affiliate_click — revenue intent
  • email_captured — list growth
  • scrolled_full_result — depth-of-engagement
  • trust_opened — dealer-trust proxy

What "good" looks like

By week 2 you should see ≥ 100 analysis_run events with ≥ 30 unique users. By week 4 the ratio share_link_copied / analysis_run should be > 8% — that's the proxy for whether HonestCost spreads on its own.
